### Ex Situ – Open-source spatial index of displaced cultural artifacts

Ex Situ is an open-source spatial index that maps over 100,000 museum artifacts from their origin sites to Western institutions, currently indexing eight collections including the Met, V&A, and SMB Berlin. Built with Next.js, Deck.gl, and Python ETL, it functions as a connector between archives, allowing researchers to download provenance data while keeping images as source URLs to avoid duplicating institutional data. The project, started as a 2022 MA thesis, is AGPL-3.0 licensed and seeks community contributions for scaling and additional museum API integrations.

- [Ex Situ – Open-source spatial index of displaced cultural artifacts](https://exsitu.app/map) — *exsitu.app*

## ⚖️ AI & Copyright

### Sony Music files second copyright suit against Udio over 30,117 recordings used for AI training

Sony Music Entertainment filed a second copyright infringement lawsuit against AI music company Udio, alleging that 30,117 sound recordings were copied without permission to train its AI models. The action follows a June 29 court ruling that denied Sony's bid to add those recordings to its original case. Sony seeks statutory damages up to $150,000 per work and an injunction, while Udio has licensed with other major labels but not Sony.

## 🚀 Software Performance

### Why l (new runtime for k and q)

The author created "l," a new runtime for the K and Q languages, rebuilding the execution engine without altering the language itself. It is built on four core ideas: SIMD as a default primitive, automatic parallelism, transparent fusion to eliminate intermediate allocations, and first-class runtime compression. This enables existing K/Q code to efficiently utilize modern hardware with multiple cores, GPUs, and specialized execution units.

- [Why l (new runtime for k and q)](https://lv1.sh/blog/why-l/) — *lv1.sh*

### 30 to 70 PRs a Day: How We Managed to Not Wreck Our Systems

Honeycomb Engineering more than doubled peak weekday merges from ~30 to ~74 by mid-2026, with AI-attributed lines rising to 82.6% of new code, while incidents grew linearly. The gains came in three phases—slow experimentation, tooling-driven adoption, and a delegation step-change after Opus 4.6—and the team kept systems stable by focusing on continuous delivery, fast CI, observability, and feature flags. The core lesson is that AI amplifies existing practices, making high-autonomy teams faster without lowering quality standards.

- [30 to 70 PRs a Day: How We Managed to Not Wreck Our Systems](https://www.honeycomb.io/blog/30-70-prs-day-how-we-managed-not-wreck-systems) — *honeycomb.io*

### A PHP server that can handle 10x as many concurrent requests as Nginx+PHP-fpm

Qbix Server is a pure PHP web server that replaces the nginx + php-fpm stack, using process forking and copy-on-write memory sharing to handle up to 1,600 concurrent PHP requests on 8GB RAM compared to ~160 with php-fpm, reducing per-worker memory from 30–60MB to ~5MB and eliminating bootstrap overhead. While its static file throughput is 56–73% of nginx, it dramatically improves PHP scalability and includes built-in WebSocket, access-controlled file streaming, and intelligent caching, with the option to place nginx or a CDN in front.

- [A PHP server that can handle 10x as many concurrent requests as Nginx+PHP-fpm](https://github.com/Qbix/webserver/blob/main/README.md) — *github.com*
